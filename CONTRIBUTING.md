# 參與貢獻指南 (Contributing to UI/UX Case Study)

很高興您對 **UI/UX Case Study** 專案感興趣！以下是參與貢獻的一些方式與規範。

## 我可以如何貢獻？

### 回報 Bug

- 在回報之前，請先搜尋 [Issues](https://github.com/your-username/ui-ux-case-study/issues) 確認是否已有重複的回報。
- 如果沒有找到相關 Issue，請 [開啟一個新的](https://github.com/your-username/ui-ux-case-study/issues/new/choose)。
- 請提供清晰的標題、詳細的描述、重現步驟以及預期行為。如有截圖或影片會更有幫助。

### 建議新功能或優化

- 所有功能建議都透過 [GitHub Issues](https://github.com/your-username/ui-ux-case-study/issues) 進行追踪。
- 在提案之前，請確認該想法是否已被討論過。
- 說明為什麼這個優化對 UI/UX 設計師或開發者是有價值的。

### 提交合併請求 (Pull Requests)

- Fork 專案並從 `main` 分支建立您的開發分支。
- 使用 `pnpm install` 安裝相依套件。
- 確保您的程式碼風格符合現有規範（我們使用 Vue 3, TypeScript 與 Tailwind CSS）。
- 提交訊息請遵循標準格式（例如：`feat:`, `fix:`, `docs:` 等）。
- 提交 PR 後，請耐候審核與反饋。

## 環境設定

```bash
# 複製專案
git clone https://github.com/your-username/ui-ux-case-study.git

# 進入目錄
cd ui-ux-case-study

# 安裝相依套件 (請使用 pnpm)
pnpm install

# 啟動開發伺服器
pnpm dev
```

## 社群規範

- 請保持尊重與互助。詳見我們的 [行為準則](CODE_OF_CONDUCT.md)。

感謝您協助我們打造更好的 UI/UX 學習資源！
